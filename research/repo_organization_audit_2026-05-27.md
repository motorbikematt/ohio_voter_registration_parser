# Repo organization audit — 2026-05-27

Scope: deep scan of `D:\vibe\election-data` to surface clutter, duplication, stale state, and gitignore drift that hurt readability for a human scanning the GitHub view. Findings are grouped by severity. Each "Action" is a concrete move/delete; nothing has been changed yet.

## Severity 1 — visible on GitHub, low-effort cleanup

These show up at the top of the GitHub file browser and shape first-impressions of the project.

**Root-level patch scripts (tracked).** `apply_cache_and_mem_fix.py` and `apply_groupings_fix.py` are one-shot migration scripts whose target patches landed in `voter_data_cleaner_v2.py` and `jurisdictional_groupings.py` long ago. They are not imported anywhere and have no `__main__` guard reason to live alongside the engine. Action: move both to `docs/archive/migrations/` (so the diff history is preserved), or `git rm` outright since the changes are already in the engine files.

**`command_history.log` (tracked, 142 lines).** Antigravity CLI session transcript from 2026-05-26. Not referenced by any code, not regenerated by the pipeline, contains nothing reusable. `.gitignore` lists `logs/` but not this file. Action: `git rm command_history.log` and add `command_history.log` to `.gitignore`.

**`main.py` (tracked, 5 lines).** Stub left from `uv init` (`print("Hello from election-data!")`). Not invoked anywhere. The grep hit for `import main` in `jurisdictional_groupings.py` is a docstring reference to the local `main()` function, not the module. Action: `git rm main.py`.

**`OhioVoterRegParser.code-workspace` (untracked, ignored by `*.code-workspace`).** Fine where it is; flagged only because it appears in the local listing.

**README is out of date.** README §"What's in this repo" lists seven scripts at root (`precinct_unc_export.py`, `precinct_party_export.py`, `export_unc_targets.py`, `voter_lookup.py`, `mixed_lean_predictor.py`, `run_lean_predictor_all_cohorts.py`, `run_mixed_lean_predictor_all_counties.py`) that actually live under `tools/` and `tools/scoring/`. README also references `requirements.txt` which does not exist — the project is on `pyproject.toml` + `uv.lock`. Action: rewrite the table to reflect the current `tools/` layout and replace the `pip install -r requirements.txt` block with `uv sync`.

## Severity 2 — SiteDesign/ is a duplicate of docs/

`SiteDesign/` is committed and parallel to `docs/`, with these issues:

- `SiteDesign/assets/v2.css` is stale relative to `docs/assets/v2.css` (24,374 vs 24,974 bytes; docs version is newer).
- `SiteDesign/assets/v2.js` is stale relative to `docs/assets/v2.js` (68,353 vs 69,604 bytes; docs version is newer).
- `SiteDesign/assets/tokens.css`, `v2-tokens.css`, `v2-hex-map.js` are byte-identical to the `docs/assets/` copies.
- `SiteDesign/v2.html` is byte-identical to `docs/v2.html`.
- `SiteDesign/manifest.json` differs from `docs/manifest.json` and is the older copy.
- `SiteDesign/Precincts.info site design.zip` (28 KB) is the source archive whose contents are presumably already extracted into the assets.

This setup invites editing the wrong copy and produces dashboard regressions. Action: delete `SiteDesign/assets/`, `SiteDesign/v2.html`, `SiteDesign/manifest.json`, and the `.zip`. Keep the three design markdowns (`DESIGN_HANDOFF_1..md`, `DESIGN_SYNTHESIS.MD`, `HANDOFF_LLM_NARRATIVE_DEMO.md`) and move them to `docs/research/design/` so all narrative and design handoffs live in one tree.

`CLAUDE.md` and `GEMINI.md` overlap heavily — both prescribe the >150-line patch-script rule, AST validation, polars-not-pandas, etc. Action: consolidate into a single `AGENTS.md` at root (the emerging convention) and `git rm GEMINI.md`. Trim `CLAUDE.md` to a one-line pointer at `AGENTS.md`, or rename `CLAUDE.md` → `AGENTS.md` outright and have `GEMINI.md` (if it must exist for the Antigravity tool) symlink or re-pointer.

## Severity 4 — .gitignore drift

The current `.gitignore` is acting as both an ignore file and a delete-list, and it has accumulated dead rules.

**Self-ignore.** Line 74 ignores `.gitignore` itself. This is a no-op (the file is already tracked) but signals confusion. Action: remove.

**Ignored-but-tracked drift.**
- `tests/` is ignored (line 28), but `tests/test_voter_data_cleaner.py` is tracked. The other four test files at `tests/test_pass_b_*.py` are not tracked. Effect: pytest runs locally but CI/contributors see only one test. Action: remove `tests/` from `.gitignore` and add all real test files via `git add tests/test_*.py`. Move `tools/test_jurisdiction_collisions.py` → `tests/` while you're there (it is the only `test_*.py` outside `tests/`).
- `docs/archive/` is ignored (line 81), but ten files under `docs/archive/` are already tracked. Either un-track them (`git rm --cached docs/archive/*.md docs/archive/memory/*.md`) and rely on the local-only convention from `tools/archive_state.py`, or remove the gitignore rule and accept the archive being public. Pick one; the current state is incoherent.

**Stranded rules for files that have moved.** Lines 66–78 ignore individual filenames (`JUPYTER_QUICKSTART.md`, `RUN_LOCALLY.md`, `LEGEND_EXPLORATION.md`, `LEGEND_AND_TOOLTIP_STRATEGY.md`, `CHARTJS_PLUGINS_RESEARCH.md`, `GEMINI_backup_20260509_2355.md`, `HANDOFF_OPUS_PASS_B.md`, `docs/research/CLAUDE_md_review.md`, `docs/research/PIPELINE_GUIDE.md`). The first six no longer exist at the root paths these rules target. The last two are in `docs/research/` — explicit ignore for tracked-but-personal docs. Action: prune the rules whose target files no longer exist at the path.

**Empty `output/` directory.** Not in `.gitignore`, not tracked (git ignores empty directories), but it sits in the listing. Action: `rmdir output` or `.gitkeep` it with a `README` if it's a planned write target.

**`.antigravitycli/0adc5338-e10a-4698-81a0-2b85b55b97aa.json` is a broken symlink.** Tracked, but I/O errors on read. Action: `git rm .antigravitycli/0adc5338-e10a-4698-81a0-2b85b55b97aa.json` and add `.antigravitycli/` to `.gitignore` (matches the `.claude/` and `.vscode/` pattern).
